## 🧠 What Is This?

**DetectAI Super Champion** is a full-stack AI content detection platform that determines whether text or images were generated by AI or written/created by a human. It combines statistical NLP analysis with LLM-based reasoning (via Groq's ultra-fast inference) for high-confidence verdicts.



---

## ✨ Features

| Feature | Description |
|---|---|
| 📝 **Text Detection** | Paste any text (≥ 20 words) for AI probability analysis |
| 🌐 **URL Detection** | Scrape and analyze content from any public URL |
| 📄 **File Upload** | Upload `.txt`, `.md`, `.html`, or `.pdf` files |
| 🖼️ **Image Detection** | Upload JPG, PNG, or WebP to detect AI-generated visuals |
| 📊 **Statistics Dashboard** | Aggregated scan results, trends, and breakdowns |
| 📋 **Scan History** | Browse, inspect, and delete past scans |
| 🔒 **Authentication** | Clerk-powered sign-in/sign-up with JWT-secured API routes |
| ⚡ **Real-time Analysis** | Powered by Groq LPU — results in under 2 seconds |

## 📝 AI Text Detection Pipeline

> How raw pasted text is turned into a high-confidence AI/Human verdict.

```mermaid
flowchart TD
    A([📋 User pastes text\n≥ 20 words required]) --> B["🔐 Auth Check\nClerk JWT verified"]
    B --> C["🧹 Pre-processor\nStrip whitespace · Normalize unicode\nTruncate to 8,000 chars"]

    C --> D["📊 Statistical NLP Engine"]
    C --> E["🤖 LLM Engine — Groq\nLlama 3.3 70B"]

    subgraph D["📊 Statistical NLP Engine"]
        D1["Burstiness score\nMeasures sentence length spikes"]
        D2["Vocabulary richness\nType-token ratio analysis"]
        D3["Transition density\nCounts filler/connector phrases"]
        D4["Sentence variance\nStdDev of sentence lengths"]
        D5["AI vocabulary scoring\nFlags over-used AI word patterns"]
    end

    subgraph E["🤖 LLM Engine — Groq · Llama 3.3 70B"]
        E1["Pass 1 — Structural analysis\nHeading patterns, paragraph rhythm"]
        E2["Pass 2 — Linguistic pattern scan\nClichés, hedging, filler phrases"]
        E3["Pass 3 — Confidence scoring\nFinal probability estimate"]
    end

    D --> F{{"⚖️ Weighted Ensemble Scorer\nStatistical weight: 40%\nLLM weight: 60%"}}
    E --> F

    F --> G["🏷️ Signal Breakdown\nai_flags · human_flags · flagged phrases"]
    G --> H["🏁 Final Verdict\nAI · Human · Mixed\nai_prob (0–100) · confidence level · summary"]
    H --> I[("💾 Persisted to SQLite\nscans table")]
```

## 🖼️ AI Image Detection Pipeline

> How an uploaded image is analyzed for signs of AI generation.

```mermaid
flowchart TD
    A([🖼️ User uploads image\nJPG · PNG · WebP]) --> B["🔐 Auth Check\nClerk JWT verified"]

    subgraph E["⚡ Groq — Llama 4 Scout Vision Analysis"]
        E1["Over-smoothing detection\nAbnormally smooth skin/textures"]
        E2["Lighting inconsistencies\nUnnatural shadow or highlight sources"]
        E3["Symmetry anomalies\nUnrealistic facial/structural symmetry"]
        E4["Compression artifact scan\nGAN-specific frequency artifacts"]
        E5["Edge coherence check\nBlurry or melted boundary regions"]
    end

    E --> F["📊 Probability Scorer\nCombines all visual signals\ninto a single ai_prob score"]
    F --> G["🏁 Final Verdict\nAI-Generated · Real / Human\nai_prob (0–100) · summary + signal list"]
    G --> H[("💾 Persisted to SQLite\nscans table · source_type = image")]
```

## 🚀 Quick Start (Local Development)

### Prerequisites

- **Node.js** ≥ 18
- **Groq API Key** — [Get one free at console.groq.com](https://console.groq.com)
- **Clerk Keys** — [Get from clerk.com](https://clerk.com)

### 1. Clone & Install

```bash
git clone https://github.com/SAMPRIT-NANDI/DETECT-AI-SUPER-CHAMPION.git
cd DETECT-AI-SUPER-CHAMPION
npm run install:all
```

### 2. Configure Environment

**`backend/.env`**
```env
PORT=4001
GROQ_API_KEY=gsk_your_key_here
FRONTEND_URL=http://localhost:5174
CLERK_PUBLISHABLE_KEY=pk_test_your_key
CLERK_SECRET_KEY=sk_test_your_key
```

**`frontend/.env`**
```env
VITE_CLERK_PUBLISHABLE_KEY=pk_test_your_key
```

### 3. Start Development Servers

```bash
npm run dev
# Backend → http://localhost:4001
# Frontend → http://localhost:5174
```

## 🔌 API Reference

> All `/api/detect/*` routes require a valid Clerk JWT in the `Authorization: Bearer <token>` header.

### Text Detection
```http
POST /api/detect/text
Content-Type: application/json

{ "text": "Your text here (≥ 20 words)..." }
```

### URL Detection
```http
POST /api/detect/url
Content-Type: application/json

{ "url": "https://example.com/article" }
```

### File Detection
```http
POST /api/detect/file
Content-Type: multipart/form-data

file: <upload>    # .txt, .md, .html, .pdf — max 10 MB
```

### Image Detection
```http
POST /api/detect/image
Content-Type: multipart/form-data

file: <upload>    # .jpg, .jpeg, .png, .webp — max 10 MB
```

### History & Stats
```http
GET    /api/history?page=1&limit=20   # Paginated scan history
GET    /api/history/:id               # Single scan detail
DELETE /api/history/:id               # Delete a scan
GET    /api/history/stats             # Aggregated statistics
GET    /api/health                    # Health check
```

## 🗄️ Database Schema

SQLite managed by `sql.js` — pure JavaScript, zero native dependencies, works serverlessly.

**`scans` table**

| Column | Type | Description |
|---|---|---|
| `id` | TEXT (UUID) | Primary key |
| `created_at` | DATETIME | Timestamp |
| `source_type` | TEXT | `text`, `url`, `file`, or `image` |
| `source_ref` | TEXT | URL or filename |
| `input_text` | TEXT | First 500 chars of input |
| `word_count` | INTEGER | Total word count |
| `verdict` | TEXT | `AI`, `Human`, or `Mixed` |
| `ai_prob` | REAL | Probability score (0–100) |
| `confidence` | TEXT | `Low` / `Medium` / `High` / `Very High` |
| `summary` | TEXT | Human-readable analysis summary |
| `signals` | TEXT | JSON — signal category scores |
| `ai_flags` | TEXT | JSON — AI-pattern indicators |
| `human_flags` | TEXT | JSON — human-pattern indicators |
| `phrases` | TEXT | JSON — flagged phrases |
| `ip_address` | TEXT | Client IP address |

## 🔒 Security

| Control | Detail |
|---|---|
| Rate limiting | 30 requests / minute per IP on `/api/detect` |
| File size cap | 10 MB maximum upload |
| Text truncation | Input capped at 8,000 characters |
| CORS | Restricted to configured `FRONTEND_URL` |
| HTTP headers | Secured via Helmet middleware |
| Auth | All API routes protected with Clerk JWT verification |
